================================================================================
                    PROJECT SUMMARY: AMBIENT EXPENSE AGENT
================================================================================

1. WHAT IS THIS PROJECT?
------------------------
This project implements an automated, event-driven expense-approval agent using 
the new ADK 2.0 Graph Workflow API (built on google-adk >= 2.0.0). It processes 
incoming expense report events, parses their contents, and enforces the following 
corporate rules:

- Under $100: Auto-approves the expense instantly in Python (no AI calls).
- $100 or More: Queries an AI model (gemini-3.1-flash-lite) to review the expense 
  for risk factors, then pauses for manual human review (Human-in-the-Loop).

2. STEPS PERFORMED TO BUILD IT
------------------------------
- Refactored project directory structure from app/ to expense_agent/.
- Setup dynamic configurations inside expense_agent/config.py.
- Configured local environment variables and dual auth mechanisms in .env.
- Built a Workflow graph in expense_agent/agent.py using Nodes and Edges:
  * parse_expense_node: Decodes input event data (plain JSON or base64).
  * route_expense_node: Directs routing based on amount threshold.
  * auto_approve_node: Emits instant approval for small expenses.
  * risk_review_node: Evaluates policy risks using the Gemini model.
  * human_approval_gate: Yields RequestInput to pause for manual decisions.
  * outcome_node: Formats and logs the final result.
- Integrated mock objects for local testing and ran passing test suites (pytest).
- Validated workflow resumption manually via structured FunctionResponse events.

3. TESTING EXAMPLES & HOW TO RUN THEM
-------------------------------------

A. EXAMPLE 1: AUTO-APPROVAL FLOW (Expenses < $100)
--------------------------------------------------
This flow bypasses the AI model entirely and automatically approves the expense.

- Step 1: Open the Web UI Playground: http://127.0.0.1:18081/dev-ui/?app=expense_agent
- Step 2: Paste the following JSON payload into the chat input:
```json
{
  "amount": 45.50,
  "submitter": "alice@example.com",
  "category": "Meals",
  "description": "Lunch meeting with client",
  "date": "2026-06-18"
}
```
- Step 3: Hit Send.
- Expected Outcome: The agent immediately returns a completed status showing:
  * Status: Approved
  * Reason: "Amount $45.50 is under threshold of $100.0. Auto-approved."
  * Risk Assessment: "Skipped (Auto-approved)"

B. EXAMPLE 2: HUMAN-IN-THE-LOOP FLOW (Expenses >= $100)
-------------------------------------------------------
This flow performs an AI risk evaluation and pauses until a human approves/rejects it.

- Step 1: Open the Web UI Playground.
- Step 2: Paste the following JSON payload into the chat input and send:
```json
{
  "amount": 150.0,
  "submitter": "bob@example.com",
  "category": "Travel",
  "description": "FirstClassFlightTicket",
  "date": "2026-06-18"
}
```
- Step 3: Wait for the agent to print its risk assessment and prompt you for a decision:
  "EXPENSE APPROVAL REQUIRED: ... Please reply with 'Approve' or 'Reject'..."
- Step 4: Type "Approve" in the chat box and send.
- Expected Outcome: The workflow resumes and outputs the final summary:
  * Status: Approved
  * Reason: "Decision made by human reviewer: Approve"
  * Risk Assessment: (The text generated by Gemini, explaining that first-class airfare for $150 might be policy-violating or misleading).

SECURITY RULES (applied BEFORE the dollar threshold check):
------------------------------------------------------------
- If SSN is detected        → scrubbed to [REDACTED SSN], escalated to human
- If Credit Card is detected → scrubbed to [REDACTED CREDIT CARD], escalated to human
- If Prompt Injection found  → "Prompt Injection" added to violations, escalated to human
- All violations appear in redacted_categories list
- Risk message format: "CRITICAL RISK: Violations detected — <list>. Escalated for human review."
- Only when NO violations found does the dollar threshold apply (auto-approve vs LLM review)

Required fields: amount, submitter, category, description, date
